# IFF 低空目标敌我识别模块说明

本文档说明 `IFF/` 目录下的实时敌我识别实验。该模块参考论文《基于改进 D-S 证据理论融合策略的低空目标敌我识别方法》（牛晓伟等）实现，但它是 DBN-TOPSIS 项目中的工程化旁路实验，不直接参与主项目的威胁度排序。

## 1. 本项目 IFF 流程大纲

### 1.1 这个模块做什么

IFF 模块用于判断低空目标相对于“最小风险返场通道”的身份倾向。输出类别为：

- `FR`：我方
- `AC`：假定我方
- `ST`：假定敌方
- `FO`：敌方
- `Theta`：不确定质量

当前 IFF 结果只写入 `IFF/results/`，不会回写 DBN-TOPSIS 主流程，也不会改变 `scores/posteriors/rank` 等威胁评估结果。

### 1.2 主要文件

- `IFF/iff_scene.py`：生成 IFF 实验场景。当前包含 7 个攻击目标和 3 个返场目标。
- `IFF/run_iff_demo.py`：运行实时识别 demo，维护每个目标的滑动窗口，导出 JSON/CSV。
- `IFF/visualizer_3d_matplotlib_export_animation.py`：生成三维动态可视化。
- `IFF/ds_iff_2026/config.py`：通道基准、偏差阈值、窗口长度、模糊隶属度和冲突权重等配置。
- `IFF/ds_iff_2026/normalize.py`：计算目标相对通道的原始偏差和归一化偏差。
- `IFF/ds_iff_2026/fuzzy.py`：模糊化、专家规则推理、重心法解模糊。
- `IFF/ds_iff_2026/bpa.py`：将模糊输出转换为 BPA，并执行收缩-膨胀修正。
- `IFF/ds_iff_2026/conflict.py`：计算 J-S 距离、D-S 冲突系数、综合冲突和折扣系数。
- `IFF/ds_iff_2026/ds_fusion.py`：Dempster 证据融合。
- `IFF/ds_iff_2026/recognizer.py`：串联完整实时识别流程。

### 1.3 运行方式

在项目根目录运行：

```powershell
python IFF\run_iff_demo.py
```

输出文件：

```text
IFF/results/iff_realtime_results.json
IFF/results/iff_realtime_results.csv
```

可视化：

```powershell
python IFF\visualizer_3d_matplotlib_export_animation.py --format gif --step 10 --fps 8
```

默认输出：

```text
IFF/results/iff_3d_iff_animation.gif
```

### 1.4 数据如何进入 IFF

场景状态由 `iff_scene.py` 生成，每个目标状态包含位置、速度、高度、航向等字段。`state_to_observation()` 将状态转换为识别器输入：

```python
Observation(time, target_id, name, H1, V, C, H2=None, truth=None)
```

当前字段含义：

- `H1`：雷达高度，来自状态中的 `Height`，单位转换为 m。
- `V`：目标速度，来自状态中的 `Speed`，按 `Mach 1 = 1224 km/h` 转换。
- `C`：航向角，来自状态中的 `Heading`。
- `H2`：ESM 高度，当前场景不生成，保持 `None`。
- `truth`：实验标签，只用于结果对照，不参与算法。

### 1.5 识别主流程

主流程由 `LowAltitudeIFFRecognizer.identify()` 串联：

```text
目标滑动窗口
  -> 计算 H1/V/C/H2 相对最小风险通道的偏差
  -> 将偏差归一化到模糊论域
  -> 对 H1/V/C 进行高斯隶属度模糊化
  -> 按专家规则得到 FR/AC/ST/FO 的模糊输出
  -> 重心法解模糊并归一化
  -> 转换为含 Theta 的 BPA
  -> 计算窗口内证据之间的 J-S 距离和 D-S 冲突系数
  -> 计算综合冲突、alpha 权重和 phi 折扣系数
  -> 对 BPA 折扣并执行收缩-膨胀修正
  -> D-S 顺序融合窗口内证据
  -> 选择最大 BPA 对应类别作为最终 label
```

### 1.6 当前场景

`create_iff_targets()` 生成 10 个目标：

- T01-T07：攻击类目标，使用 `DirectedAttackTarget`，会朝我方编队未来位置机动。
- T08-T10：返场/IFF 目标，使用 `IFFRouteTarget`，沿返场通道附近飞行。

`generate_friendly_series()` 生成 4 架我方编队飞机，主要供攻击目标瞄准和三维可视化使用。

## 2. 与论文中 IFF 方法的不同之处

本代码按论文的主流程实现：航迹参数标准化、模糊推理、重心法解模糊、BPA 构造、J-S 距离与 D-S 冲突联合度量、证据折扣、收缩-膨胀修正、D-S 融合。但当前工程实现与论文仍有以下差异。

### 2.1 场景来源不同

论文围绕最小风险返场通道中的低空目标识别问题展开，目标参数来自论文表格中的典型低空目标样本。

本代码不是直接复现论文表 2 的 5 类目标，而是在 DBN-TOPSIS 工程中构造实时动态场景：

- 7 个攻击目标来自本项目自定义三维攻防场景。
- 3 个返场目标用于补充 `FR/AC/ST` 类身份样本。
- 场景是连续时序仿真，而不是论文表格中的离散测试样本。

### 2.2 数据来源不同

论文明确使用雷达和电子支援措施 ESM 获取低空目标航迹参数，其中输入包括：

- 雷达高度 `H1`
- 目标速度 `V`
- 磁航向 `C`
- ESM 高度 `H2`

当前代码只在场景中稳定生成 `H1/V/C`，`H2` 保持 `None`。虽然 `normalize.py` 和数据结构保留了 `H2` 字段，但实际推理规则没有使用 `H2`。

### 2.3 输入维度和规则数量不同

论文算法设计使用 4 个输入变量 `H1/V/C/H2`，并为每个输入构造多级模糊子集。论文中的规则规模对应四输入组合。

当前代码为了工程可控性，简化为：

- 输入变量：`H1/V/C` 三个。
- 模糊等级：低/中/高 3 级。
- 专家规则：`3 * 3 * 3 = 27` 条，写在 `fuzzy.py` 的 `EXPERT_RULE_TABLE` 中。

因此当前实现保留了论文“模糊推理 -> 解模糊 -> BPA”的方法链路，但规则表不是论文完整规则表。

### 2.4 模糊隶属度配置不同

论文使用高斯隶属度函数描述输入变量与输出变量之间的模糊关系。

当前代码也使用高斯隶属度函数，但中心点和尺度是工程配置：

- `LEVEL_CENTERS = (0.0, 0.5, 1.0)`
- `fuzzy_input_sigma = 0.125`
- `fuzzy_output_sigma = 0.125`

这些参数用于当前三等级规则表，并未逐项校准到论文表 3、表 4 的数值。

### 2.5 通道基准与场景参数不同

论文仿真实验中的通道飞行规则和低空目标参数来自论文设定。

当前代码中的通道基准由 `RouteProfile` 给出，并且 `run_iff_demo.py` 可覆盖默认配置。例如当前 demo 使用：

```python
RouteProfile(height_m=5000.0, speed_kmh=600.0, heading_deg=290.0)
```

这更像工程仿真通道，而不是论文原始表格通道的逐项复现。

### 2.6 证据来源的含义不同

论文中的多组证据主要来自不同传感器/不同航迹参数在多个时刻下的识别结果。

当前代码的融合证据来自同一目标最近若干时刻的滑动窗口：

```python
window = track[-config.window_size:]
```

默认窗口长度为 3。也就是说，本项目更偏“实时连续时序融合”，而不是静态表格样本融合。

### 2.7 综合冲突计算是工程化版本

论文提出将 J-S 散度和 D-S 冲突系数结合，计算综合冲突并据此进行证据折扣。

当前代码实现为可配置加权：

```python
综合冲突 = js_weight * js_distance + ds_weight * ds_conflict
```

默认：

```python
js_conflict_weight = 0.50
ds_conflict_weight = 0.50
```

然后再根据证据间支持度计算 `alpha` 和 `phi` 折扣系数。这保留了论文思想，但具体工程公式不是论文公式的逐符号复刻。

### 2.8 收缩-膨胀函数的使用方式不同

论文使用收缩-膨胀函数动态调整 BPA，以突出高置信类别并削弱冲突证据影响。

当前代码在 `discount_and_refine_masses()` 中先做证据折扣，再对折扣后的 Mass 执行 `contraction_expansion()`。函数形式遵循论文思想，但阈值、归一化和调用时机按当前工程实现组织。

### 2.9 结果验证方式不同

论文给出了表 3、表 4、表 5 等数值结果，并与 Yager 合成公式、证据折扣法等多种方法比较。

当前代码主要输出：

- 每个目标每个时刻的 `FR/AC/ST/FO/Theta`
- 当前偏差 `Delta_H1/Delta_V/Delta_C/Delta_H2`
- 滑动窗口平均冲突 `mean_conflict`
- 三维 GIF/MP4 可视化

当前还没有自动化脚本逐项复现论文表格，也没有与论文对照方法做批量数值对比。

### 2.10 命名略有调整

论文中常见记号可理解为：

- `RF` / `FR`：我方
- `CA` / `AC`：假定我方
- `TS` / `ST`：假定敌方
- `OF` / `FO`：敌方

代码统一使用 `FR/AC/ST/FO/Theta`。

## 3. 仍需改进的点

以下按重要程度排序。

### 1. 补齐 ESM 高度 H2 并接入四输入推理

当前最大差距是没有实际生成和使用 `H2`。论文方法强调雷达与电子支援措施共同提供航迹参数，`H2` 是低空雷达盲区补充信息的重要来源。

建议：

- 在 `iff_scene.py` 中生成 ESM 高度观测或模拟 ESM 误差。
- 在 `state_to_observation()` 中填充 `H2`。
- 将 `fuzzy.py` 从三输入 `H1/V/C` 扩展为四输入 `H1/V/C/H2`。

### 2. 恢复论文规模的模糊规则体系

当前 27 条规则是工程简化版。若目标是论文复现，应补齐论文的完整专家规则表。

建议：

- 明确每个输入变量的等级数。
- 将规则表从当前三输入三等级扩展为论文对应的四输入规则。
- 将规则来源写成可审计表格或配置文件，而不是隐含在代码里。

### 3. 用论文表格做回归测试

当前没有自动验证代码是否能复现论文表 3、表 4 的 BPA 和融合结果。

建议：

- 新增 `tests/` 或 `IFF/tests/`。
- 固化论文表 2 的输入数据。
- 对比初始 BPA、折扣后 BPA、最终融合结果。
- 设置合理容差，避免后续改参数时无意破坏论文复现能力。

### 4. 明确区分“论文复现模式”和“项目动态场景模式”

当前 README 和代码容易混淆两件事：

- 按论文表格复现实验。
- 在 DBN-TOPSIS 动态三维场景中运行 IFF 旁路。

建议：

- 增加 `paper_reproduction.py`：只跑论文表格数据。
- 保留 `run_iff_demo.py`：只跑动态场景。
- 两者共享算法包 `ds_iff_2026/`，但输入数据和评估指标分开。

### 5. 校准通道基准和场景单位

当前代码中通道高度、目标高度、可视化高度、Mach/km/h 转换共同影响输出。参数分散在 `run_iff_demo.py`、`config.py` 和 `iff_scene.py` 中。

建议：

- 统一说明单位：位置 km，高度 m/km 转换，速度 Mach/km/h 转换。
- 将通道配置集中到一个地方。
- 明确 demo 使用的通道是否要贴近论文，还是服务三维可视化。

### 6. 改进综合冲突和折扣公式的可追溯性

当前实现保留了 J-S + D-S 冲突和折扣思想，但公式写法是工程化版本。

建议：

- 在 `conflict.py` 中用注释标注每一步对应论文公式。
- 保存中间矩阵：J-S 矩阵、D-S 冲突矩阵、综合冲突矩阵、`alpha`、`phi`。
- 增加单元测试验证边界情况，例如完全一致证据、完全冲突证据、单证据窗口。

### 7. 完善目标真值与评估指标

当前 T01-T07 没有 IFF 真值，T08-T10 有 `FR/AC/ST` 真值，但没有覆盖 `FO` 返场目标。

建议：

- 为动态场景补充 `FO` 或伪装敌方返场目标。
- 为每个目标明确期望标签或期望区间。
- 输出准确率、混淆矩阵、Theta 均值、冲突均值等评估指标。

### 8. 整理编码和文档一致性

部分源文件中文注释存在编码损坏，旧 README 也曾出现乱码。

建议：

- 统一文件编码为 UTF-8。
- 清理损坏注释。
- 将论文差异、参数含义、运行入口和结果字段保持同步更新。
